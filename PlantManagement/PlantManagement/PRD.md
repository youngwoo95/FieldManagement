# PlantManagement PRD (프로젝트 기반)

- 버전: 0.2 (초안)
- 작성일: 2026-04-28
- 제품명: PlantManagement (WPF 데스크톱)
- 작성 기준: 현재 저장소 구현 코드

## 1. 제품 개요

PlantManagement는 제조/설비 운영 관리를 위한 Windows 데스크톱 애플리케이션이다. 현재 아래 6개 모듈을 중심으로 동작한다.

1. 대시보드
2. 고객사 관리
3. 설비 관리
4. 수주 관리
5. 작업 현황 관리
6. 설비 배치 현황

아키텍처는 MVVM + DI 구조이며, 데이터 흐름은 아래처럼 혼합 상태다.

1. 실데이터 연동: 고객사, 설비, 공지
2. 로컬/목 데이터: 수주, 작업 현황, 층별 설비 배치

## 2. 문제 정의

현장 운영팀은 고객사/설비/수주/작업 정보를 한곳에서 관리하고, 설비 상태를 빠르게 확인할 수 있어야 한다.  
하지만 정보가 분산되면 아래 문제가 반복된다.

1. 최신 상태 확인 지연
2. 수기 관리로 인한 데이터 불일치
3. 작업 이력 추적 어려움

## 3. 목표

1. 운영 핵심 데이터를 단일 앱에서 관리한다.
2. 조회/입력에 드는 시간을 줄인다.
3. 향후 확장 가능한 서비스/저장소 구조를 유지한다.

## 4. 비목표 (현 단계)

1. 웹/모바일 클라이언트 제공
2. MES/ERP/IoT 실시간 외부 연동
3. 전 생산 프로세스 자동화

## 5. 대상 사용자

1. 생산/운영 관리자
2. 설비/보전 담당자
3. 고객사/수주/작업 등록 담당자

## 6. 범위

### 6.1 포함 범위 (현재 코드 기준)

1. 6개 메뉴 네비게이션 및 화면 전환
2. 다크/라이트 테마 전환
3. 대시보드 차트 및 공지 조회
4. 고객사 CRUD(추가/수정/소프트 삭제) + 검색
5. 설비 CRUD(추가/수정/소프트 삭제) + 검색
6. 수주 목록/검색/추가 + PDF 미리보기 패널
7. 작업 현황 목록/검색/추가 + PDF 미리보기 패널
8. 층별 설비 배치 편집 + 캔버스 마커 표시 + 로그
9. 예외 로그 파일 저장

### 6.2 제외 범위 (미구현)

1. 수주 DB 연동 CRUD
2. 작업 현황 DB 연동 CRUD
3. 설비 배치(층/장비 매핑) DB 영속화
4. 사용자 권한/역할 관리
5. 감사 이력 화면 및 고급 리포팅

## 7. 핵심 사용자 시나리오

### 7.1 고객사 담당자

1. 고객사 관리 진입
2. 키워드 검색(고객사명/담당자/연락처)
3. 고객사 정보 추가 또는 수정
4. 선택 항목 삭제(소프트 삭제)

### 7.2 설비 담당자

1. 설비 관리 진입
2. 설비명/제조사 기준 검색
3. 설비 추가 또는 수정
4. 선택 항목 삭제(소프트 삭제)

### 7.3 운영 담당자

1. 수주/작업 현황 화면 진입
2. 모달 창으로 신규 등록
3. 선택 행의 첨부 PDF 즉시 확인

### 7.4 설비 배치 담당자

1. 설비 배치 현황 진입
2. 층 선택 후 마커 배치 확인
3. 편집 창에서 층-설비 매핑 수정 후 반영

## 8. 기능 요구사항 (FR)

### FR-01 메뉴 및 화면 전환

1. 시스템은 6개 메뉴를 제공해야 한다.
2. 앱 시작 시 기본 화면은 대시보드여야 한다.

### FR-02 테마 전환

1. 사용자는 다크/라이트 테마를 전환할 수 있어야 한다.
2. 테마 전환은 앱 재시작 없이 즉시 적용되어야 한다.

### FR-03 대시보드

1. 시스템은 대시보드 차트를 표시해야 한다.
2. 공지사항은 서비스/리포지토리를 통해 조회되어야 한다.

### FR-04 고객사 관리

1. 고객사 추가/수정/삭제가 가능해야 한다.
2. 수정은 단일 선택 기준으로 동작해야 한다.
3. 다중 선택 삭제가 가능해야 한다.
4. 검색어 기준 필터링을 제공해야 한다.
5. 삭제는 `delYn` 기반 소프트 삭제여야 한다.

### FR-05 설비 관리

1. 설비 추가/수정/삭제가 가능해야 한다.
2. 수정은 단일 선택 기준으로 동작해야 한다.
3. 다중 선택 삭제가 가능해야 한다.
4. 검색어 기준 필터링을 제공해야 한다.
5. 삭제는 `delYn` 기반 소프트 삭제여야 한다.

### FR-06 수주 관리 (현재 구현 상태)

1. 수주 목록 조회 및 검색이 가능해야 한다.
2. 수주 추가 모달을 제공해야 한다.
3. 선택된 수주의 PDF 미리보기를 제공해야 한다.
4. 파일이 없으면 안내 메시지를 표시해야 한다.

### FR-07 작업 현황 관리 (현재 구현 상태)

1. 작업 현황 목록 조회 및 검색이 가능해야 한다.
2. 작업 추가 모달을 제공해야 한다.
3. 선택된 작업의 PDF 미리보기를 제공해야 한다.
4. 파일이 없으면 안내 메시지를 표시해야 한다.

### FR-08 설비 배치 현황 (현재 구현 상태)

1. 층 목록과 층별 마커 배치를 보여줘야 한다.
2. 보기/편집 모드 전환이 가능해야 한다.
3. 편집 창에서 층-설비 매핑을 수정할 수 있어야 한다.
4. 사용자 동작 로그를 화면에 누적 표시해야 한다.

### FR-09 로깅

1. 시스템은 예외 발생 시 로그를 파일로 저장해야 한다.
2. 저장 경로는 `SystemLog/YYYY/MM` 구조를 따라야 한다.

## 9. 비기능 요구사항 (NFR)

1. 플랫폼: Windows 데스크톱 (.NET `net10.0-windows`, WPF)
2. 아키텍처: MVVM + DI
3. 데이터 접근: EF Core + Dapper 혼합
4. 반응성: 일반 사무용 PC에서 메뉴 전환 체감 1초 이내
5. 안정성: 일부 데이터 로드 실패 시 앱 전체 크래시 방지
6. 유지보수성: ViewModel/Service/Repository 계층 분리 유지

## 10. 데이터 및 연동 요구사항

1. `PlantContext` 기반 MySQL 스키마를 사용한다.
2. 고객사/설비/공지는 DB 연동 상태다.
3. 수주/작업/설비 배치는 DB 영속화 구현이 필요하다.
4. 운영 배포 전 연결 문자열은 코드 외부 설정으로 분리해야 한다.

## 11. KPI

1. 고객사/설비 등록 및 수정 처리 시간 단축
2. 수주/작업 조회 소요 시간 단축
3. 필수 입력값 누락률 감소
4. 런타임 오류 발생 건수(로그 기준) 감소

## 12. 리스크 및 제약사항

1. 실데이터와 목데이터 혼재로 사용자 혼선 가능성
2. 대시보드 일부 수치가 샘플 값이라 신뢰도 저하 가능성
3. 소스 내 DB 인증정보 포함은 보안 리스크
4. 일부 문자열/인코딩 정리 필요

## 13. 단계별 릴리즈 계획 (제안)

### 1단계: 현재 기능 안정화

1. 입력 검증/오류 메시지 표준화
2. 검색/필터 조건 일관성 정리
3. 예외 처리 및 로그 확인성 강화

### 2단계: 데이터 연동 완성

1. 수주 모듈 DB CRUD 구현
2. 작업 현황 모듈 DB CRUD 구현
3. 설비 배치(층/장비) DB 저장 구조 구현

### 3단계: 운영 고도화

1. 설정/시크릿 외부화
2. 역할/권한 모델 도입
3. 감사 이력 및 리포트 기능 추가

## 14. MVP 수용 기준

1. 6개 메뉴가 정상 전환된다.
2. 고객사/설비 CRUD가 DB 기준으로 정상 동작한다.
3. 대시보드 공지 로드 실패가 앱 시작 실패로 이어지지 않는다.
4. 수주/작업 추가 및 PDF 미리보기/예외 안내가 정상 동작한다.
5. 설비 배치 편집 결과가 화면에 즉시 반영된다.
6. 예외 로그가 지정 경로에 기록된다.

## 15. 오픈 이슈

2. 작업 상태값 표준(`InProgress` 등) 정의 필요
3. 운영 KPI 목표치 및 보고 주기 확정 필요
